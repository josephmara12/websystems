Reflection

I’ve always loved a minimalist, simple and neat design. A great example of a company who thrives using this model is Apple. I aimed to recreate this with my website. There isn’t too much going on with a wide use of negative space, which gives off a ‘cleaner’ look. My webpage is made up of 4 pages all linked to the same css stylesheet in order to provide a consistent feel. I chose to try something different putting the Navbar on the left side of the page rather than the top. There is also no obvious header, as it is essentially ‘invisible’ using just text instead.

Webpage Structure



Technical

Requirements Used For
Header tag The header tag was used to create the invisible header, which consisted of a h1 element of text at the top of the page
Nav(anchor tag) The achcor tags were used for navigation and is how I created the navigation bar. When used with the href tag it allowed me to hyperlink the other 3 pages as per the requirements.
#box class The box class is used to style all the rounded boxes used in my webpages. The class descibes the width, height and colour. I had to embed the positioning of each box however as the boxes needed to be in specific locations
Table The table element has obviously been used to explain the different technical elements of this webpage
CSS class The .class Selector was used to style the Navbar, ensuing it was consistent in all pages. It was also used to style the background of the comments page.
CSS ID The .ID selector was one of my most used selectors and I tended to implement them more than whole classes as I had many individual items which needed to be positioned rather than a group of elements. For example all the rounded boxes were styled using ID selectors. The reason I didn’t use a class for these was because I needed to position them differently.
Div The Div tag was used to split my page into seperate sections. Basically the div tag was used for the Navbar, the main body and the footer. This made styling much easier in terms of positioning as elements were grouped together.
Span The span tag was used to style the header of each page ensuring there is consistency among pages with the same font


Aesthetics

Font

The font I used for this webpage was Trebuchet MS. It is a humanist sans serif and looks simple and clean, highlighting the intutitive nature of my website.

Colour Scheme

I chose to employ a seemingly basic colour scheme using light blues and greys. However, the simplicity is the look I was aiming to achieve. The contrast between the light grey (rgb 219, 214, 214, 0.767) and solid blue (#1e95da) allows users to clearly see content and naviagte around the page.

Accessibility

Intuitive

I ensured that this website was extremely intuitive allowing any type of user to access all pages and navigate easily around them. Elements that added to this include the ability to hover over links in the navbar, the simplicity of design and the contrasting colour scheme.

Design (Images)

All my images are given an alt> tag so that if the image fails to load user can still see what is meant to be there, giving context to users if not accessible. The images themselves help break the page into sections allowing users to clearly see and read content, essentially not bunching all the words on the page together

Structure and design

As mentioned above I have designed my website in a way that makes it extremely intuitive for users. This is helped by the simple colour scheme which won’t effect the visually impaired. The contrast between bold blues and light greys really makes certain elements stand out ensuing content is easy to read. The boxes around text are also labeled and draws attention to that segment.
